# Brief 3 — Stage 4: SEND Terminology Alignment

**Version**: 1.0  
**SEND CT**: 2025-09-26 (Package 60)  
**Implementation Guide**: SENDIG v3.1  

## Purpose

Maps all 25 cross-domain concordance entries from the Brief 3 concordance map to CDISC SEND controlled terminology. This alignment layer enables the concordance engine to query real SEND datasets by translating prose-level concordance patterns into SEND variable codes.

## Implementation Guidance

### Query Pattern

- 1. MI domain: WHERE MISTRESC IN (mapped_nonneo_terms) AND MISPEC = mapped_organ
- 2. LB domain: WHERE LBTESTCD IN (mapped_test_codes) — compare vs concurrent control
- 3. OM domain: WHERE OMTESTCD = mapped_weight_code — compare vs concurrent control
- 4. CL domain: WHERE CLSTRESC LIKE mapped_observation_terms
- 5. Compute concordance tier based on domain coverage count

### Synonym Resolution

- Build lookup from mi_finding_alignment.terms where is_synonym=true
- Normalize all MISTRESC to preferred terms before evaluation
- Split comma-separated qualifiers: match base term + qualifier separately
- Case-insensitive matching mandatory
- Log unresolved terms for curation

### Severity Threshold

- **Exclude**: MINIMAL (likely background in rodents)
- **Include**: MILD and above
- **Strong signal**: MARKED or SEVERE — supports concordance even without full domain coverage

### Cross-Study Challenges

- ~32% of MISTRESC values in SENDIG v3.1 submissions are non-standard (Carfagna 2021)
- UNREMARKABLE and NORMAL are not in NONNEO codelist but are most frequent MISTRESC values
- Sponsor-defined extensions common for cell-type qualifiers
- Engine should maintain a synonym/mapping table that grows with each new study processed
