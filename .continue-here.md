## Active Bug: Pattern Classification Regression

### What's broken

`classify_dose_response()` in `backend/services/analysis/classification.py` uses adjacent-group pairwise steps to classify patterns. This fails for **gradual dose-response curves** where each adjacent step is small but the cumulative effect is large.

### Concrete example

BASO Males in PointCross:
- Control: 0.050, D1: 0.037, D2: 0.024, D3: 0.023
- Effect size: d = -1.17 (large effect!)
- Each adjacent step (~0.013) falls within the equivalence band (0.016)
- System classifies as **"flat"** — should be **"threshold_decrease"**

The same flaw causes false **"non_monotonic"** when a minor wobble between adjacent groups reverses direction, even though the control-relative trajectory is clearly monotonic.

### Root cause

`_classify_from_steps()` (line 134) only examines adjacent-group steps. It never checks the overall control-relative trajectory. When a dose-response distributes change gradually across 3+ dose groups, individual steps fall within the noise band even though the total deviation from control is well beyond it.

### Key code locations

- `backend/services/analysis/classification.py:127` — `_step_direction()` compares adjacent means
- `backend/services/analysis/classification.py:134` — `_classify_from_steps()` maps step sequence to pattern
- `backend/services/analysis/classification.py:209` — `classify_dose_response()` main entry point
- `_EQUIVALENCE_FRACTION = 0.5` and `_pooled_sd()` determine the band width

### Fix approach

Add a **control-relative override** after adjacent-step classification:

1. After computing steps and initial pattern, check if result is "flat" or "non_monotonic"
2. Compute each treated group's deviation from control: `means[i] - means[0]`
3. If ALL treated deviations that exceed the band point in the same direction, override to `threshold_increase` or `threshold_decrease`
4. For "non_monotonic": if control-relative deviations are all same-direction (even if adjacent steps wobble), override to `threshold_*`

This preserves existing logic for clear signals and only activates for gradual/noisy cases.
