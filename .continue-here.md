## What's Broken

BW Males (and some other endpoints) classified as `non_monotonic` when they should be `threshold_decrease`. Females for the same timepoints are correct.

Example — BW M 2nd timepoint:
- means: [364.07, 370.53, 365.8, 319.13]
- Control→D1: +6.46 (noise), D1→D2: -4.73, D2→D3: -46.67
- Band = 0.5 × pooled_SD = 6.32
- Step 0→1 barely exceeds band (6.46 > 6.32) → classified "up"
- Steps: [up, flat, down] → `non_monotonic` (WRONG — should be `threshold_decrease`)

## Regression Point

**Commit `7e70de9`** (2026-03-03) — "feat: time course pane — switch from % change to Hedges' g effect size"

Backend consistency fix in that commit: passed `last_dosing_day` to `compute_bw_findings()` for recovery animal treatment-period pooling. This correctly changed which animals are included in BW stats, but shifted group means slightly:

The data change is correct (bug fix). The classifier is fragile at the band boundary.

## Root Cause

`_pooled_sd()` in `classification.py:95` computes pooled SD from **treated groups only** (excludes control). The high-dose group has artificially low SD (8.62 vs ~15-16 for others) because treatment compresses variability. This deflates the pooled estimate and makes the band too tight.

- Pooled SD (treated only): 12.65 → band = 6.32
- Pooled SD (all groups incl. control): 13.60 → band = 6.80
- With all-groups band: step 6.46 < 6.80 → flat → threshold_decrease (CORRECT)

## Impact of Including Control in Pooled SD
