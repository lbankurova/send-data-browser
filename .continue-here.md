## Decisions made this session

- **Autoresearch insight:** constraints > capabilities. Enforcement through mechanical gates, not prose instructions.
- **Validation ratchet is nuanced:** degradation routes to research, not rollback. Binary keep/discard is wrong for scientific apps.
- **Decision log seeded from distill audit:** 28 patterns extracted from project corpus.
- **Density over size caps:** no arbitrary token limits on research. Enforce density (every paragraph must contain new info), not length.
- **Corpus load before research:** Step 0 reads existing knowledge to prevent restatement.
- **Context = RAM, disk = storage:** agents must not rely on "remembering" — re-read from disk at every step.
- **Three-signal context meter:** re-read detection (diagnostic), rolling volume (prognostic), working set breadth (cognitive).
- **Sweep is mandatory, not partial:** every step runs. ROADMAP moved to Step 2 (highest impact). Spec-archive backlinks prevent stale ROADMAP entries.
- **Pre-commit hooks prevent stale indexes at source** (root cause), sweep cleans up what slips through (symptom).
- **Coverage facts auto-regeneration:** wired into /lattice:review (on engine change) and /ops:sweep (freshness check).
