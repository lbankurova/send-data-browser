## Decisions made this session

### Cohort View architecture
- **Rail = grouping toggles** (All/Histopath/Deaths/Syndromes), not subject roster. Dose group cards expand to show subjects. Same as Findings rail.
- **Center top = always two-column split.** Left: scatter or findings landscape. Right: BW spaghetti or subject-vs-group.
- **Subject Findings Landscape** replaces multi-track timeline. Y=endpoints sorted by signal, X=days, color=severity. 65/35 chart-to-table split.
- **NOAEL overlay** as subject table columns (finding count + role + LOO tipping count).
- **No empty state.** Auto-select first rail card.
- **Saved cohorts = contextual actions** (right-click scatter, pin in context panel), not rail toggle.
- **PK exposure:** table columns, Exposure tab, exposure-response scatter toggle.
- **Study design guards:** crossover/escalation = hard gate, small-N = suppress scatter, multi-compound = rail partitioning.
- **Consume existing outlier scores** (Hamada, Mahalanobis, RCV, LOO). Don't reinvent.
- **RCV significance overlay** in landscape for per-subject LB statistical significance.
- **Per-domain exclusion rules** — respect existing, don't add new. Per-endpoint, not per-subject.
- **Correlation matrix** in context panel.
- **Food efficiency toggle** alongside BW in landscape augmented rows.

### Similarity engine (from research)
- **Gower distance** — only metric handling mixed types natively at small N with missing-data support.
- **Non-metric MDS** — deterministic, stable, handles non-Euclidean Gower distances.
- **Complete-linkage** — valid on arbitrary dissimilarities (Ward is not).
- **Feature vector consumes existing pipeline outputs** (fold-change, Hedges' g, severity). Does NOT recompute from raw data.
- **Temporal trajectory similarity deferred** — preclinical data too thin (2-3 LB timepoints, terminal MI/MA). Base engine uses terminal/worst-case snapshots.
